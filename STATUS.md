### How This File Works

Items flow: **ideas/INBOX.md -> ideas/PIPELINE.md -> Concerns / Work -> Watch -> Archive** (or AGENTS.md Hard Rules).

- **ideas/INBOX.md** is for fast capture of loose user ideas.
- **ideas/PIPELINE.md** is for triage, deduplication, and promotion.
- **Concerns** are raw capture. One line, instant, no analysis.
- **Work** is the ordered task board and dispatch surface. Position is priority. Each item has a **Files** column (what it writes — the collision boundary) and a **Depends** column (what must be done first, including read dependencies). **Status** tracks dispatch/landing only; it does not by itself mean "currently verified." Current trust lives in **Verify**. See AGENTS.md Parallel Dispatch for claiming rules.
- **Watch** holds items worth keeping visible after landing. Legacy rows without explicit freshness labels should be read as historical until refreshed.
- **Archive** is append-only history with explanations.
- **Verify / Trust labels** should start with one of: `current:`, `historical:`, `contradicted:`, or `unknown:`. If a claim depends on runtime or test evidence, include the date and environment.
- If a user idea is not being executed in the current turn, make sure it lands in `ideas/INBOX.md` or `ideas/PIPELINE.md` instead of only in chat.

PLAN.md changes require user approval. When behavior contradicts a PLAN.md assumption, add it here as a Concern - do not implement the conflicting approach.
When current evidence contradicts an older Verify or Watch claim, rewrite or downgrade the claim immediately instead of leaving stale certainty in place.

Nothing is ever lost. Nothing is ever deleted.
