Research notes, refreshed 2026-05-02:

- Lemon Amiga lists Scorched Tanks as Amiga OCS/ECS shareware, written in AMOS, with v1.85 as a common public reference (`https://www.lemonamiga.com/game/scorched-tanks`).
- Amiga Boing's gameplay note confirms the expected flow: title screen, player setup, buying weapons, then a generated world with angle/power shots and sound effects (`https://amigaboing.net/2022/04/28/scorched-tanks/`).
- GameBase Amiga Update 91.3 says Scorched Tanks was fixed to run on Kickstart 1.3, 1 MB, OCS. The extracted GameBase config also points to Kickstart v1.3, OCS, 68000, no fast RAM (`https://www.t2e.pl/News/gamebase-i-kolekcje.450/gamebase-gamebase-amiga-1.7-seya-update-93.0-24112025.24916`).
- The GB64 GameBase Amiga update thread repeats the same fix note: Scorched Tanks now runs on Kickstart 1.3, 1 MB, OCS (`https://www.gamebase64.com/forum/viewtopic.php?start=150&t=6015`).
- The extracted GameBase MDB also has a diagnostic WHDLoad extra for Scorched Tanks v1.90: `cd=dh1:scorchedtanks; whdload=generickick31.slave custom=scorchedtanks PRELOAD;`. The referenced `ScorchedTanks190_WHDL.zip` was not present in the 91.3 update archive extracted locally.
- WHDLoad's official GenericKick install can run programs through WHDLoad KickEmu, but its GenericKick31 slave expects real Kickstart 3.1 images and GenericKick13 expects a real Kickstart 1.3 image (`https://whdload.de/apps/GenericKick.html`).
- WHDLoad's requirements documentation says KickEmu installs require original Kickstart images, extracted from real hardware or sourced from Cloanto Amiga Forever where supported (`https://www.whdload.de/docs/en/need.html`).
- Cloanto/Amiga Forever's RP9 listing and Game Pack I FAQ list Scorched Tanks as an Amiga Forever-playable/included game. This is a potential licensed entitlement path for both firmware and game media, but it still requires a license/integration and is not bundled in this branch (`https://rp9.com/9.106`, `https://www.amigaforever.com/kb/15-124`).
- A Reddit /r/amiga discussion independently mentions a generic WHDLoad route for Scorched Tanks on AGA hard-disk setups, which matches the GameBase MDB hint but is not sufficient provenance by itself (`https://www.reddit.com/r/amiga/comments/n1yv9c/tanx_amiga_a_tank_game_from_1991_yeah/`).
- Amiga-news.de reported on 2002-12-15 that Amiga Arena released the full Scorched Tanks v1.90 with Michael Welch's permission as `STanks.lha` (`https://www.amiga-news.de/en/news/AN-2002-12-00121-EN.html`). This is a strong rights signal for a v1.90 full-version package, but the original Amiga Arena download has not yet been recovered as a provenance-clean artifact.
- A 2011 WinUAE troubleshooting thread for Scorched Tanks reports a package/config that requires Kickstart ROM v1.3 for A500/A1000/A2000 class hardware; replies correctly note the Kickstart ROM is still copyrighted and must be supplied/licensed by the user (`https://www.forum-3dcenter.org/vbulletin/archive/index.php/t-501716.html`).
- Lemon Amiga comments include a WinUAE play report and describe Scorched Tanks as OCS/ECS AMOS shareware, reinforcing that the target runtime is a normal Amiga emulator with compatible Kickstart rather than a browser remake (`https://www.lemonamiga.com/game/scorched-tanks`).
- AMOS-for-Windows, packaged by Mequa Innovations, proves AMOS Professional can run under bundled free AROS-68k in WinUAE 2.3.2 using an AGA/68040/max-speed profile, but this does not make Scorched Tanks compatible with AROS (`https://www.t2e.pl/News/prasa-i-magazyny.417/amiga-amos-professional-pod-windowsy.5008`).
- Aminet Set 3 lists `Scorch185.lha` under freely distributable `game/shoot` software (`https://wiki.aminet.net/Contents_of_Aminet_Set_3`).
- AmigaPD's v1.77 review says supported Amigas range from A500 through A4000, but incompatibilities are usually memory-related and Workbench on 512K chip RAM may crash (`https://www.amigapd.com/june-2012-reviews-by-amigapd.html`).
- The 17 Bit catalogue lists Scorched Tanks v1.77 as a shareware/public-domain catalogue item, so alternate shareware media may be worth testing if the AP41 v1.75 build remains blocked (`https://www.amigapd.com/17-bit-catalogue.html`).
- An amiga.org discussion reports Scorched Tanks v1.85 had a terrain bug and calls v1.75 the last bug-free version; another reply reports a registered v1.90-era copy still played after an `amos.library` change but sound stopped working (`https://forum.amiga.org/index.php?topic=29155.0`). This supports testing both v1.75 and v1.90 while treating sound as a required acceptance check.
- AROS 68k documentation describes broad Amiga binary compatibility as the goal, but not perfect compatibility; the current blocker is therefore tracked as an AMOS/AROS runtime compatibility failure, not a media discovery failure (`https://en.wikibooks.org/wiki/Aros/Platforms/68k_support`).
- vAmiga/vAmigaWeb ROM documentation notes that original Commodore Kickstart ROMs are not shipped, legally purchased ROMs can be supplied by the user, and AROS is freely redistributable but not compatible with all Amiga programs (`https://dirkwhoffmann.github.io/vAmiga/docs/References/Configuration/Roms.html`, `https://vamigaweb.github.io/doc/about.html`).
- Libretro PUAE documentation says the Amiga core falls back to built-in AROS when a proper Kickstart is missing, and its A500 preset is Kickstart v1.3 with OCS, 512K chip RAM, and 512K slow RAM (`https://docs.libretro.com/library/puae/`).
- Amiga.org's AMOS-for-Windows thread confirms AMOS Professional itself can run on AROS-68k in WinUAE 2.3.2, so the Scorched failure is narrower than "AMOS never runs on AROS" (`https://forum.amiga.org/index.php?topic=58632.0`).
- Amiga.org's `0x0100000F` discussion and the classic exec alert definitions both identify `AN_BadFreeAddr` as an invalid `FreeMem()` pointer or corrupted memory-list symptom, so alert suppression alone is not a valid compatibility proof (`https://forum.amiga.org/index.php?topic=24187.0`, `https://d0.se/include/exec/alerts.i`).
- Reddit Amiga emulation discussions report the practical browser/SAE/vAmiga pattern: AROS is the legal default fallback, but users need era-correct Kickstart ROM images for full compatibility (`https://www.reddit.com/r/amiga/comments/1k4v6nv/problems_with_emulation/`).
- Search did not find a public Scorched Tanks + AMOS + AROS-in-browser compatibility recipe. Browser "play online" pages exist, but they are not enough provenance for a rights-cleared Workflow package.

This page launches the autostart ADF in vAmigaWeb from a browser-only Workflow result. The parent page fetches the same-site ADF and injects the bytes into the iframe so static hosts do not need cross-origin ADF headers. The default `auto` firmware mode first checks the intentionally absent `licensed/kickstart-a500-1.3.rom` entitlement path, falls back to the open AROS ROM path when that file is not provisioned, and still accepts a user-owned Kickstart ROM through a local file picker without uploading it to the Workflow host. Bundled AROS keeps the A1200-style diagnostic profile; user/host Kickstart switches to A500 OCS 512K chip + 512K slow and no fast RAM. `?firmware=hosted-kickstart` is a strict deployment proof mode and fails loudly if the licensed ROM is missing.

Compatibility finding, verified 2026-05-02: Scorched Tanks v1.75 boots to the AP41 coverdisk menu under AROS in SAE. F2 launches the game. The LF `Stack 65536` A1200-style AROS profile reaches the Scorched Tanks v1.75 title/legal screen, but after user input or extended wait the original runtime black-screens before gameplay. A500-style AROS returns to Workbench instead of gameplay. PlanetEmu v1.77, Aminet v1.85, and GameBase fixed v1.90 all launch AMOS and then fail under AROS with `0x0100000F` memory-header/free-list corruption or black/empty-screen state before gameplay; the exact GameBase A500 512K chip + 512K slow-memory AROS profile returns to empty Workbench. Mounting a second diagnostic `ENV:` floppy stops the Workbench `ENV` requester but still opens an empty ScorchedTanks window instead of exposing game icons or starting gameplay. Scorched Tanks v0.95 runs its own startup but A500-style AROS reports not enough memory, while larger chip/fast profiles black-screen or abort in SAE memory mapping. EmulatorJS/PUAE with A500/OCS/AROS fallback reaches Workbench only or an unusable emulator menu state for the tested diagnostic ADFs. WinUAE 2.3.2 with AMOS-for-Windows AROS successfully boots AMOS Professional itself, but GameBase fixed v1.90 still raises `0x0100000F - Memory header not located`; a 4 MB chip/no-fast diagnostic hides the stack-autostart alert as a black screen and no-EndCLI confirms the same alert. Dismissing Continue leads to a black screen, not gameplay. Browser SAE reruns with the older AMOS-for-Windows AROS ROM/ext also fail earlier in reset loops: v1.77 A500 512K chip + 512K slow reset 10 times in 90s, v1.77 A1200 2 MB chip + 4 MB fast reset 21 times in 90s, and v0.95 A500 512K chip + 512K slow reset 10 times in 90s. Hash-qualified evidence files are `scorched-v177-autostart__aros-amiga-m68k-rom-9557fd50__gamebase-a500-512kchip-512kbogo-90s-console.log`, `scorched-v177-autostart__aros-amiga-m68k-rom-9557fd50__a1200-2mchip-4mfast-90s-console.log`, and `scorched-tanks-v0-95-1993-09-15-welch-michael-sw-amos__aros-amiga-m68k-rom-9557fd50__gamebase-a500-512kchip-512kbogo-90s-console.log`; screenshots are flat non-game canvases, not gameplay. A v1.77 two-disk shim that adds AMOS-for-Windows `amos.library` to `LIBS:` also fails: A500 512K chip + 512K slow reports not enough memory, and A500 1 MB chip + 2 MB fast still shows `0x0100000F - Memory header not located` (`scorched-v177-amoslib-df1-autostart__aros-amiga-m68k-rom-9fb24d10__gamebase-a500-512kchip-512kbogo-120s.png`, `scorched-v177-amoslib-df1-autostart__aros-amiga-m68k-rom-9fb24d10__v185-a500-1mchip-2mfast-120s.png`). The latest official AROS 2026-04-28 nightly ROM/ext pair matches the bundled AROS assets byte-for-byte. Community GameBase/WHDLoad recipes now point to licensed Kickstart images rather than a proven free AROS path. No licensed local Kickstart 1.3 ROM was found on this host. Exact-game acceptance remains `NOT_PLAYABLE_YET` until the original runtime reaches gameplay with input, sound, and a tank hit captured.

No proprietary Kickstart ROM is bundled. The browser-native artillery port is kept only as a labeled fallback and must not be reported by Workflow as the exact original game.
