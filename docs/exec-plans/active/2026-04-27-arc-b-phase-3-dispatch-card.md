## 6. What could go wrong

### 6.a Non-import string references the audit may have missed

The Phase 1+2 caller migration handles imports. **String references to `fantasy_author` / `_rename_compat` / `WORKFLOW_AUTHOR_RENAME_COMPAT` that don't appear in `from`/`import` statements** can survive:

- **`patch("workflow.author_server.X", ...)`** — handled in §1.3 / §4. Verified 0 in the 2026-04-29 Phase 2 replay; re-check before Phase 3 dispatch.
- **`monkeypatch.setattr(... , "rename_compat_enabled", ...)`** — handled in `tests/test_discovery.py` cleanup (§2.c).
- **`monkeypatch.setenv("WORKFLOW_AUTHOR_RENAME_COMPAT", ...)`** — only in `tests/test_rename_compat.py` (deleted) and `tests/test_discovery.py` (cleaned up). Verify with §1.4 grep at dispatch time.
- **Docstring / comment references** — e.g., codex specs may mention "via _rename_compat alias" or "during the Author rename." Non-blocking; cosmetic. Sweep separately if desired.
- **`fantasy_author` as a CLASS NAME** — `domains/fantasy_daemon/skill.py:19` defines `class FantasyAuthorDomain` (verified). This stays. Class-rename (`FantasyAuthorDomain` → `FantasyDaemonDomain`) is a separate arc, NOT in Arc B scope. Don't delete the class name in Phase 3.

### 6.b Hidden lazy imports

Phase 1+2 only catches imports that grep can see at file-read time. **Lazy imports inside try/except or inside function bodies** can survive:

```bash
# Sweep BEFORE Phase 3 dispatch
grep -rE 'importlib\.import_module\(.+fantasy_author|importlib\.import_module\(.+_rename_compat' workflow/ tests/ scripts/ --include='*.py'
grep -rE 'getattr\(.+, ["\047]fantasy_author' workflow/ tests/ scripts/ --include='*.py'
```

If any hits: investigate, migrate, fold into Phase 2 or escalate to lead.

### 6.c `domains/fantasy_author/phases/__init__.py` directory may not be empty

The prep doc lists `domains/fantasy_author/__init__.py` and `phases/__init__.py` for deletion. Verify there are NO other files in those directories before `git rm -r domains/fantasy_author/`:

```bash
find domains/fantasy_author -type f
# Expected: only the two __init__.py files (deleted explicitly above) + possibly __pycache__/ (ignored)
```

If other files exist (e.g., a stray test fixture, a doc file), surface to lead before `git rm -r`.

### 6.d Smoke-fail mode signals

If `WORKFLOW_AUTHOR_RENAME_COMPAT=0 pytest -q` fails, the failing test name typically encodes the holdout caller:

- `ImportError: No module named 'domains.fantasy_author.X'` → a Phase 2 caller migration was missed for module `X`. Locate via `grep -rn "domains.fantasy_author.X" workflow/ tests/` and migrate.
- `AttributeError: module 'workflow.author_server' has no attribute 'X'` → a `patch("workflow.author_server.X", ...)` survived. Locate via §1.3 grep + migrate.
- `NameError: rename_compat_enabled` → discovery.py edit missed something. Re-read §2.c.

Iterate until both smoke runs are green; each iteration is one targeted edit.
