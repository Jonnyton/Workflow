## §2. Load-bearing product signals

Grouped by severity. Only the critical + moderate items are dispatch-worthy; the cosmetic items are noted + held.

### §2.1 CRITICAL — address immediately or launch fails

**S-1. Chatbot-assumes-Workflow is not enforced.** LIVE-F1 + CHAT-1 are the same bug surfacing in two personas. Current `control_station` prompt either doesn't include the "ambiguous-but-plausibly-Workflow = invoke, not ask" directive, or it's drowned out by other instructions. This is the single most important product behavior for tier-1 onboarding — the zero-ceremony promise from §26 depends on it.

**S-2. Injection-hallucination cross-conversation bleed.** LIVE-F2 Yardi fabrication. Likely in #12/#15 scope (design-note §17.4 training-exclusion is structurally right, but #15 text-edit mitigations haven't landed in prod). Blocker for any financial / legal / clinical tier-1 use case.

**S-3. Confidential-tier routing must be auditable, not promised.** CHAT-2 + OBS-1. Devin's bet rests entirely on tray + chatbot agreeing that confidential data stayed local. Design note §17.4 has the structural enforcement (concept-only Postgres role). But the **user-visible verification** (tray dashboard showing concrete LLM binding, hard fail-local switch, tray ↔ chatbot consistency) is not yet specified.

**S-4. Tier-3 PR-review SLA.** SLA-1 is the single-biggest tier-3-scaling risk. Without explicit SLA in CONTRIBUTING.md + an actual maintainer cadence, tier-3 cohort dies at first-contact. Forever-rule implication: maintainer rotation is a §22 succession concern — not just host responding within 48h.

### §2.2 MODERATE — specify now, implement in MVP window

**S-5. Platform-vocabulary bleed into user chat.** LIVE-F3 ("branch", "canon", "few-shot reference" in Maya's first session). Fixable by tightening `control_station` prompt to *use user-vocabulary, not engine-vocabulary, until the user uses a term first*. Pure prompt engineering.

**S-6. Launch-day catalog must have ≥3 real editing workflows + ≥3 payables/bookkeeping workflows + ≥3 research workflows.** CHAT-1 + implicit in Maya's "can you help me do my payables" expectation. Empty catalog at launch = immediate bounce for every persona. Task #55 shipped 4 domain-realistic sample nodes; the gap is the *count* per domain + branch composites.

**S-7. Tray observability consistency.** OBS-1. Tray dashboard + chatbot must show matching run state, same LLM, same privacy tier, same ETA. This is a tray-spec amendment (task #30 / spec `2026-04-18-daemon-host-tray-changes.md`), not yet specified.

**S-8. Tier-2 tray install-trust surface.** TRAY-1 (unsigned .exe SmartScreen) + TRAY-2 (ambiguous "link chatbot" with no recovery) + TRAY-3 (privacy pane must be concrete). Packaging spec amendments; already implicit in §13.2 onboarding but needs user-facing copy specificity.
