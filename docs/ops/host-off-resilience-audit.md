## Host-dependent today — must close ✗

### HD-1 — Droplet SSH key ~~only on host's disk~~ → CLOSED: dual-key, bus factor = 2

- **Primary key:** `~/.ssh/workflow_deploy_ed25519` (private) on host's
  Windows machine; public half in `/root/.ssh/authorized_keys` on Droplet.
  Seeded as `DO_SSH_KEY` GH Actions secret.
- **Backup key (second factor):** generated by
  `scripts/bootstrap_add_second_ssh_key.py`; public half appended to
  Droplet's `/root/.ssh/authorized_keys`; private half seeded as
  `DO_SSH_KEY_BACKUP` GH Actions secret.
- **Bus factor:** 2 (either key grants SSH access independently).
- **What breaks when host is off:** Nothing — GHA can SSH the Droplet
  using `DO_SSH_KEY` or `DO_SSH_KEY_BACKUP`.
- **Setup (one-time, ~2 min):**
  ```bash
  python scripts/bootstrap_add_second_ssh_key.py \
      --host 161.35.237.133 \
      --primary-key ~/.ssh/workflow_deploy_ed25519 \
      --repo Jonnyton/Workflow
  ```
  Script is idempotent: skips if `DO_SSH_KEY_BACKUP` already seeded.
- **Recovery if primary key lost:** use `DO_SSH_KEY_BACKUP` secret;
  generate a new primary, upload to Droplet, re-seed `DO_SSH_KEY`.
- **Also recommended:** password-manager backup of both private keys.

### HD-2 — `workflow-secrets.env` only on host's disk

- **Where it lives:** `$HOME/workflow-secrets.env` on host's Windows
  machine (referenced from the in-repo `.env` pointer per
  `memory/project_secrets_outside_repo.md`).
- **Contents (redacted):** `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ZONE_ID`,
  `DIGITALOCEAN_TOKEN`, `HETZNER_EMAIL`, `HETZNER_PASSWORD`,
  `OPENAI_API_KEY` (historical; default daemons now ignore API-key provider

- **What breaks when host is off:** Any lead session on a different
  machine can't source secrets; token rotation can't happen locally.
- **Close action:** password-manager backup of the full file
  (1Password / Bitwarden / age-encrypted blob in a separate private
  repo). Not committed anywhere.
- **Also needed in GH Actions secrets:**
  - `CLOUDFLARE_API_TOKEN` — for Worker deploy + emergency DNS.
  - `DIGITALOCEAN_TOKEN` — for droplet-level actions from CI (reboot,
    resize, snapshot).

### HD-3 — ~~LLM endpoint not bound on prod daemon~~ — SUPERSEDED: subscription-only provider auth

- **Historical resolution:** codex CLI shipped in the daemon image
  (Dockerfile Node 20 + `@openai/codex`) with `OPENAI_API_KEY` seeded to
  `/etc/workflow/env` + GH Actions secret.
- **Superseded 2026-04-30:** default daemon LLM execution is subscription-only.
  `OPENAI_API_KEY` is stripped when
  `WORKFLOW_ALLOW_API_KEY_PROVIDERS` is not truthy; use subscription-backed
  Claude/Codex auth instead.
- **Why codex (not Anthropic):** codex CLI is subprocess-based (hard
  rule 3) and already wrapped by `workflow.providers.codex_provider`.
  No new HTTP wiring needed.
- **Verified:** `python scripts/mcp_probe.py status` against
  `tinyassets.io/mcp` → `"llm_endpoint_bound": "codex"`. Live on
  2026-04-21.
